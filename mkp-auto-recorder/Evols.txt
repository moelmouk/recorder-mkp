Voici une analyse des évolutions fonctionnelles qui pourraient enrichir votre extension de test automatisé :

1. Gestion des attentes et des temps d'attente
Attente conditionnelle : Ajouter des conditions d'attente personnalisables (attendre qu'un élément soit visible/cliquable/...)
Timeouts configurables : Permettre de définir des délais d'attente globaux ou par commande
Attente d'éléments dynamiques : Meilleure gestion des chargements AJAX et des éléments qui apparaissent de manière asynchrone
2. Variables et paramétrage
Variables utilisateur : Permettre de définir et d'utiliser des variables dans les scénarios
Paramètres externes : Importer des données depuis des fichiers CSV/Excel pour des tests paramétrés
Gestion des environnements : Configurations différentes selon les environnements (dev, test, préprod, prod)
3. Assertions et vérifications
Vérifications avancées : Ajouter des assertions pour vérifier le contenu, le style, les attributs des éléments
Captures d'écran : Prendre des captures d'écran à des étapes clés ou en cas d'échec
Comparaison d'images : Pour détecter les régressions visuelles
4. Gestion des données
Bases de données : Connexion à des bases de données pour vérifier les données
API Testing : Tester directement des appels API en plus des interactions UI
Mocking : Simuler des réponses API pour les tests en isolation
5. Améliorations de l'interface
Éditeur de scénarios avancé : Avec coloration syntaxique, autocomplétion
Glisser-déposer : Pour réorganiser les commandes
Historique des exécutions : Conserver un historique des exécutions avec statistiques
Rapports détaillés : Génération de rapports HTML/PDF avec captures d'écran
6. Intégrations
CI/CD : Intégration avec Jenkins, GitLab CI, GitHub Actions
Outils de test : Export vers des formats compatibles avec Selenium, Cypress, etc.
Gestion de tests : Intégration avec des outils comme TestRail, Zephyr
7. Fonctionnalités avancées
Tests cross-navigateurs : Exécution sur différents navigateurs
Tests mobiles : Support des émulateurs mobiles
Reconnaissance d'images : Pour les éléments difficiles à localiser
Reconnaissance OCR : Pour interagir avec du texte dans des images
8. Sécurité et performances
Tests de charge : Simulation de plusieurs utilisateurs
Tests de sécurité : Détection de vulnérabilités courantes
Audit de performance : Mesure des temps de réponse
9. Collaboration
Partage de scénarios : Fonctionnalité de partage entre utilisateurs
Commentaires : Ajouter des commentaires aux étapes de test
Versioning : Gestion des versions des scénarios
10. Documentation et support
Documentation intégrée : Aide contextuelle pour chaque type de commande
Tutoriels interactifs : Pour guider les nouveaux utilisateurs
Exemples de scénarios : Bibliothèque d'exemples pour différents cas d'utilisation

11-01-2026: Indicateur visuel du scénario en cours dans l’onglet Groupes
- Mise en évidence du scénario en cours d’exécution pendant la lecture d’un groupe (classe CSS `.playing` sur l’élément `.scenario-item`).
- Synchronisation en temps réel via l’état `playbackState` (ajout de `scenarioId`) et le polling du popup.
- Le surlignage se met à jour lors du changement de scénario, défile jusqu’à l’élément courant, et est nettoyé à la fin/arrêt de la lecture.

- Ajout d’une option dans Paramètres > Configuration pour définir le temps d’attente entre scénarios (ms).
- Stockage dans `chrome.storage.local` (clé `mkpInterScenarioDelayMs`), valeur par défaut 500 ms, bornes 0–600000 ms.
- Transmission au background lors du `PLAY_GROUP` et application du délai entre scénarios avec prise en compte de la pause/arrêt.
- Affichage d’un compte à rebours du délai dans l’overlay de lecture (si visible).
